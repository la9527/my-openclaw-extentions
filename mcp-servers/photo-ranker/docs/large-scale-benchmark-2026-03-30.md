# 대규모 벤치마크 결과 보고서 (2026-03-30)

## 1. 테스트 개요

| 항목 | 값 |
|------|-----|
| **고유 이미지** | 45장 (picsum.photos 랜덤 이미지) |
| **중복 변형** | 25장 (5장 × 5변형) |
| **총 이미지** | 70장 |
| **하드웨어** | Apple M4, 32GB RAM |
| **VLM 모델** | Qwen2.5-VL-7B-Instruct-4bit (mlx-vlm 0.4.2) |
| **Aesthetic** | CLIP ViT-L-14 + LAION v2.5 MLP |
| **Face** | mediapipe BlazeFace short-range 0.10.33 |
| **Dedup** | imagehash (ahash + phash) |

### 테스트 목적

1. 50장 이상 대규모 이미지에서 전체 파이프라인 안정성 검증
2. 해상도(256~1024px)에 따른 VLM 정확도·속도 비교로 최적 해상도 결정
3. 중복 검출 엔진의 recall / precision 검증
4. 각 엔진 점수 분포의 합리성 확인

### 이미지 소스 특성

picsum.photos에서 ID 분포를 고르게 하여 50장을 요청, 45장 다운로드 성공 (5장 404).
picsum은 키워드 검색을 지원하지 않아 이미지 종류 편향이 존재한다:
- **풍경/자연 위주**: outdoor 60%, portrait/birthday/graduation/celebration 부재
- 이 편향은 이벤트 분류 분포에 영향을 주지만, **해상도 비교**와 **중복 검출** 검증에는 충분한 데이터

### 2.4 결론: 최적 해상도

**512px 유지를 권장한다.**

| 근거 | 설명 |
|------|------|
| 일관성 | 256px가 92%로 가장 높지만, 12장 기준 1장 차이 (통계적 유의성 부족) |
| 속도 | 256→512 차이는 ~2초로 체감 범위, 1024는 +5초로 부담 |
| 해상도 증가 효과 없음 | 768/1024로 올려도 일관성이 512보다 높지 않음 (83% 동일) |
| 세부 인식 | 512px에서도 건물, 자연, 음식 등 주요 피사체를 충분히 인식 |
| VRAM/메모리 | 512px는 32GB Mac 환경에서 안정적 (1024는 메모리 피크 상승) |

> 만약 처리 속도가 최우선이라면 **384px**도 고려 가능 (속도 12% 향상, 일관성 83%).
> **256px 이하로 줄이는 것은 피사체 세부 손실 가능성이 있어 비권장.**

## 4. 중복 검출 정확도

### 4.1 변형 유형

| 변형 | 설명 | 예상 난이도 |
|------|------|------------|
| crop95 | 가장자리 5% 잘라냄 | 쉬움 |
| bright115 | 밝기 +15% | 쉬움 |
| jpegq40 | JPEG 재압축 (Q=40) | 보통 |
| downup50 | 50% 축소 후 원래 크기 복원 | 보통 |
| blur15 | 가우시안 블러 r=1.5 | 쉬움 |

### 4.2 Recall (검출률)

| Threshold | 검출/총 | 검출률 |
|-----------|--------|--------|
| **8 (기본값)** | **25/25** | **100%** |

> 모든 변형 유형에서 source–duplicate 쌍이 100% 검출됨.
> ahash가 crop, brightness, blur, recompression, downscale 변형에 모두 강건.

### 4.3 False Positives (오검출)

| Threshold | ahash 그룹 수 | phash 그룹 수 | 오검출 쌍 |
|-----------|-------------|-------------|----------|
| 4 | 7 | 5 | — |
| 6 | 8 | 5 | — |
| **8** | **7** | **5** | **116** |
| 10 | 6 | 5 | — |
| 12 | 6 | 5 | — |

> **threshold=8에서 오검출 116쌍**: 시각적으로 비슷한 풍경 이미지들이 같은 그룹으로 묶임.
> 원인: picsum의 유사 풍경 (산, 숲, 하늘 등)이 ahash에서 비슷한 색상 분포를 가짐.

### 4.4 개선 방안

1. **phash 우선 사용**: phash는 5그룹만 생성 → DCT 기반이라 구조적 유사성을 더 잘 구분
2. **threshold 조정**: threshold=6으로 낮추면 오검출 대폭 감소 가능 (recall 유지 확인 필요)
3. **dual-hash 전략**: ahash + phash 모두 threshold 이하일 때만 중복으로 판정
4. **현실 사진에서는 오검출 감소 예상**: picsum의 유사 풍경 편향이 오검출의 주원인
